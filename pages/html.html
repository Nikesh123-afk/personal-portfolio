Semantic Elements

Semantic HTML elements provide meaning to content, improving SEO, accessibility, and code maintainability.

Explanation

Semantic elements like <header>, <nav>, <main>, <article>, and <footer> describe their meaning, not just appearance.

  • <header> - Page or section header
  • <nav> - Navigation links
  • <main> - Primary content
  • <article> - Standalone content
  • <aside> - Sidebar content
  • <footer> - Page or section footer
Working Example
<main>
  <article>
    <header>
      <h1>Article Title</h1>
      <time datetime="2025-01-15">January 15, 2025</time>
    </header>
    <section>
      <p>Article content here...</p>
    </section>
    <aside>
      <p>Related information</p>
    </aside>
  </article>
</main>
Annotation: This structure is meaningful to screen readers, search engines, and developers.

Forms & Inputs

Modern HTML5 forms provide built-in validation, accessibility features, and enhanced user experience.

Explanation

HTML5 introduced new input types, validation attributes, and accessibility features:

  • Input types: email, tel, url, date, number
  • Validation: required, pattern, min/max
  • Accessibility: labels, ARIA attributes
  • User experience: placeholder, autocomplete
Working Example

Semantic HTML5 Elements

Why Semantic Elements Matter

Semantic elements provide meaning to the content they contain, making it easier for browsers, search engines, and assistive technologies to understand the structure and purpose of web content.

Benefits:
  • SEO Improvement: Search engines better understand content structure
  • Accessibility: Screen readers can navigate content more effectively
  • Code Readability: Developers can easily understand document structure
  • Maintenance: Easier to maintain and update code

Semantic Structure Example

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Semantic HTML5</title>
</head>
<body>
    <header>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section id="home">
            <h1>Welcome</h1>
            <article>
                <h2>Article Title</h2>
                <p>Article content...</p>
            </article>
        </section>
        
        <aside>
            <h3>Related Links</h3>
        </aside>
    </main>
    
    <footer>
        <p>© 2025 Website</p>
    </footer>
</body>
</html>

HTML5 Semantic Elements

<header>

Represents introductory content or navigation links

Used for page headers, article headers, or section headers
<nav>

Defines navigation links within the document

Main navigation, breadcrumbs, or pagination
<main>

Contains the primary content of the document

Should be unique and not contained within other semantic elements
<section>

Represents a standalone section of content

Chapters, tabs, or thematic groupings
<article>

Self-contained, independently distributable content

Blog posts, news articles, or user comments
<aside>

Content related to but separate from main content

Sidebars, callout boxes, or related links
<footer>

Contains footer information for its section

Copyright, contact info, or related documents
<figure>

Self-contained content with optional caption

Images, diagrams, code listings

HTML5 Form Features

Input Validation

Built-in validation for email, URL, number, and required fields without JavaScript.

<input type="email" required>
Date & Time Inputs

Native date, time, and datetime-local pickers for better user experience.

<input type="date">
<input type="time">
Range Controls

Slider inputs for numeric ranges with min, max, and step attributes.

<input type="range" min="1" max="10" step="1">
Color Picker

Native color picker for easy color selection.

<input type="color">
Search Input

Specialized search input with enhanced styling and behavior.

<input type="search" placeholder="Search...">

Accessibility & Best Practices

ARIA Labels

Accessible Rich Internet Applications (ARIA) attributes provide semantic information to assistive technologies.

Example:
<button aria-label="Close dialog">Ã—</button>

Screen Reader Support

Proper heading hierarchy, alt text for images, and descriptive link text help screen readers navigate content.

Example:
<img src="chart.png" alt="Sales increase of 25% in Q4">

Keyboard Navigation

Ensure all interactive elements are accessible via keyboard navigation with proper tab order.

Example:
<a href="#main" class="skip-link">Skip to main content</a>
